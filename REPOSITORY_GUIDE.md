# REPOSITORY_GUIDE.md — Avian Observatory

> A map of the repository for future contributors (and future me). What each
> top-level folder and standalone script is for, its category, when to use it,
> and whether it's expected to keep changing.
>
> **Categories:** `runtime` (needed for the app to run) · `training` · `evaluation`
> · `publication` · `archive` (frozen history) · plus `data`, `docs`, `deploy`,
> `deprecated`/`dead` where those are clearer.
>
> Deeper detail lives in [ARCHITECTURE.md](ARCHITECTURE.md),
> [PROJECT_STATE.md](PROJECT_STATE.md), and [DEPLOYMENT.md](DEPLOYMENT.md).

## Orientation — "I want to…"

| Goal | Go to |
|---|---|
| Change API behaviour | `api/` (routers → services → repositories) |
| Change the science (inference, stats, detection) | `birddash/` |
| Change the web UI | `frontend/` |
| Add a DB column/table | `alembic/` (new migration) + `api/models.py` |
| Re-train a model | `train_*.py`, `train_birdnet_embeddings.py`, `preprocess.py` |
| Re-run evaluation / regenerate metrics | `evaluate_v5.py`, `regenerate_cnn_evaluation.py` |
| Regenerate paper figures/tables | `generate_publication_assets.py` → `build_asset_registry.py` |
| Deploy | `Dockerfile`, `heroku.yml`, [DEPLOYMENT.md](DEPLOYMENT.md) |

## Top-level folders

| Folder | Category | Purpose | When to use | Changes in future? |
|---|---|---|---|---|
| `api/` | **runtime** | FastAPI backend — routers → services → repositories, DTOs, settings, migrations entrypoints. | Always (the platform runtime). | **Yes** — active development. |
| `birddash/` | **runtime** | Framework-free scientific core: `audio`, `birdnet`, `detection`, `nt_model`, `results`, `metrics`, `statistics`, `taxonomy`, `config`. Imported by the API **and** the Streamlit app. | Any change to inference, detection, statistics, or species naming. | **Yes** — active; keep it free of Streamlit/FastAPI imports (test-enforced). |
| `frontend/` | **runtime** | Next.js web app (dashboard, map, recording workspace, species, biodiversity, models, About). | Any UI change. Read `frontend/AGENTS.md` first (modified Next.js). | **Yes** — active. |
| `alembic/` | **runtime** | Database migrations (`env.py` + `versions/`), run by the release phase. | When the schema changes — add a new migration; never edit an applied one. | **Yes** — additive (new versions only). |
| `tests/` | **evaluation** (dev) | Golden parity, detection-parsing, and API tests. `generate_golden.py` regenerates goldens for intended model changes. | Run before commits; extend with new features. | **Yes** — grows with the code. |
| `docs/` | **docs** | `METHODOLOGY.md` (cited stats), `PUBLICATION_ASSETS.md` (asset registry), demo charts. | Writing up methods / Paper 1. | **Yes** — occasionally. |
| `evaluation/` | **evaluation + publication** | `registry.json`, `asset_registry.json`, `original/cnn_*`, `reproduced/v5`, `paper1/` (figures/tables), plus historical thesis PNGs at the root. **The API reads `registry.json` at runtime.** | Regenerated by the evaluation/publication scripts; read by `/models/registry`. | **Yes** — regenerated when models are re-evaluated. Root `*.png` are frozen history. |
| `models/` | **runtime + archive** | Model binaries (gitignored). Production `NT_Bird_BirdNET_Classifier.tflite` + `nt_bird_cnn_best.keras` are runtime; the other `.keras` variants (v2/v3/v4 backups, `final`, `finetuned`) are research history. | Present at runtime for inference; produced by training. | **Rarely** — only on retrain. |
| `training_data/` | **training + data** | Xeno-canto dataset (gitignored audio) + `dataset_metadata.csv`. **The CSV is runtime** (comparison ground truth); the audio is training-only. | Training/eval; the CSV is read by the model comparison. | **Rarely** — on dataset refresh. |
| `sample_audio/` | **runtime (data)** | Demo recordings seeded into the DB and served for playback/inference. | Always (seeded). | **Occasionally** — when demo recordings change. |
| `birdnet_results2/` | **runtime (data)** | Global-BirdNET result CSVs (`config.BIRDNET_RESULTS_DIR`), read by comparison/biodiversity/detections. | Always (read at runtime). | **Rarely**. |
| `detections/` | **runtime (data)** | Cached v5.2 SED JSON, NT-prediction JSON, spectrogram PNGs (gitignored), plus the committed `validation/` set. | Read/written at runtime; regenerated by the detector. | **Yes** — regenerated per analysis. |
| `spectrograms/` | **mixed** | `label_map.json` = runtime (CNN labels + species seed); `*.npy` = training (preprocessed, gitignored). | The JSON is always needed; the arrays only for CNN training. | **Rarely**. |
| `archive/` | **archive** | `model_iterations/` — superseded detector scripts, retained for reproducibility. **Referenced by nothing outside `archive/`.** | Historical reference only. | **No** — frozen. |
| `birdnet_v5_results/`, `birdnet_custom_results/` | **evaluation (artefacts)** | Raw v5-classifier CSVs from past runs. **Not read at runtime.** | Debugging / forensic reference. | **No** — stale artefacts. |
| `sample_audio_old/` | **archive (data)** | Legacy demo audio; **not seeded, not read at runtime**. | Historical only. | **No**. |

## Notes for contributors

- **`birddash/` must stay framework-free** (no Streamlit/FastAPI imports — a test enforces it). It's the one place shared by the API and the Streamlit app.
- **"Not imported by the runtime" ≠ "deletable."** The `train_*`/`download_*`/`evaluate_*` scripts are essential for model retraining and research reproducibility.
- **Evaluation is reproducible, not hand-edited.** Never type metrics into docs; run the evaluation/publication scripts and let `verify_metric_provenance.py` confirm consistency.
- **Deploy config** (`Dockerfile`, `.dockerignore`, `heroku.yml`, `Procfile`, `app.json`) is prepared but nothing is deployed — see [DEPLOYMENT.md](DEPLOYMENT.md).
- **Two dead files** (`config.py`, `tests/_reference.py`) reference only each other; consider them together if cleaning up.
